## The Problem

Most digital learning tools are built by developers — not learning designers. The result? Platforms that deliver content efficiently but fail to actually teach. No feedback loops. No cognitive load management. No sense of progress. Learners click through and forget.

EduPulse was built to challenge that assumption.

---

## The Solution

EduPulse is a browser-based interactive learning dashboard that applies core instructional design principles — directly inside the UI. It's not a course platform. It's a **demonstration**: what happens when every design decision — from layout to interaction to feedback — is made with learning science in mind.

Built on **React 18** (no JSX — plain `createElement` for zero-build-step deployment), EduPulse shows how progress tracking, feedback loops, and data visualisation can be embedded into a digital product without adding complexity — by reducing it.

---

## Key Features

- **📊 Progress Tracking** — Learners always know where they are, what they've completed, and what comes next. No guesswork, no cognitive overload from ambiguity.

- **🔁 Feedback Loops** — Every interaction returns meaningful information. Responses aren't just "correct" or "incorrect" — they guide the learner toward understanding *why*.

- **📈 Data Visualization** — Learning state is surfaced visually. Patterns in engagement and progress become visible, both for the learner and for anyone interpreting the experience.

- **🧠 Cognitive Load Reduction** — Content is chunked, sequenced, and scaffolded using instructional design principles — so learners aren't overwhelmed, they're guided.

- **⚡ State-Driven UI Architecture** — The entire experience responds dynamically to learner behaviour. No page reloads. No broken flows. Just a smooth, responsive learning journey.

---

## The Design Thinking Behind It

EduPulse wasn't built features-first. It was designed outcomes-first — the same way any good instructional design project should be.

The question it started with: *"If you could make a digital product behave like a well-designed lesson, what would that actually look like in code?"*

That question shaped every component:
- Progress bars aren't decoration — they're motivation mechanics
- Feedback messages aren't error states — they're teaching moments
- State management isn't just technical architecture — it's the equivalent of a learning pathway

## Tech Stack

| Layer | Tech |
|-------|------|
| Frontend Framework | React 18 (UMD build — no JSX, no bundler) |
| Rendering | `React.createElement` — plain JS, runs directly in browser |
| Styling | CSS (external stylesheet) |
| Typography | Playfair Display · DM Sans · JetBrains Mono |
| Hosting | GitHub Pages |
| Design Approach | Instructional Design · Learning Experience Design · UX |

> **Why no JSX or bundler?** By using React's UMD build with plain `createElement`, EduPulse runs directly in any browser with zero build tooling — making it lightweight, portable, and easy to fork or inspect.
